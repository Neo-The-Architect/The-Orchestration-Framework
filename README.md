# The Orchestration Framework

> **Status:** v0.2 — public release. Methodology stable. Templates evolving. v0.2 introduces fundamentals discipline (philosophy/07), security layer for foreign code (architecture/), knowledge architecture as compounding infrastructure (architecture/), and the anonymization protocol for public-private artifact separation (architecture/).

The Orchestration Framework is the core IP for AI-native business operations: a knowledge architecture under which a single human operator commands an agentic stack of AI tools through a structured, version-controlled file system. It is not a productivity system, a note-taking method, or a workflow guide — it is a working constitution for operators who treat AI agents as conductors, executors, and observers operating against a persistent, file-based knowledge layer that represents the operator's life, business, and infrastructure as a single unified system. The operator does not perform the work; the operator orchestrates the system that performs the work. Files are the medium, agents are the labor, velocity is the outcome — and the framework itself is the compounding asset: every capability the operator learns lands in the architecture and becomes available to every future engagement.

This is methodology and reference, not a starter kit. The Template Repository feature is intentionally disabled. You read the philosophy, you copy the template skeletons into your own private vault on your own machine, and you make the framework yours.

## Quick start

1. **Read the philosophy** — start with [`philosophy/`](philosophy/). The six foundational principles explain why the framework is shaped the way it is. Without them, the architecture and rhythms read as arbitrary rules.
2. **Review the architecture** — [`architecture/`](architecture/) defines the vault structure, naming conventions, and read/write permissions across the agent stack. This is the canonical part of the methodology that adopters preserve as-is.
3. **Instantiate the templates** — copy files from [`templates/`](templates/) into your own private vault. Do not fork this repo and work in it; the public repo is reference material, not a place for your data.
