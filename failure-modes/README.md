# Failure Modes

Every framework fails. The question is whether failure is observable and recoverable. The Orchestration Framework anticipates seven recurring failure modes and specifies a countermeasure for each.

The single document [`seven-failure-modes.md`](seven-failure-modes.md) catalogs all seven with their symptoms and countermeasures. Read it once during initial adoption, then return to it during the [monthly system audit](../rhythms/monthly.md) — the seven failure modes are the diagnostic checklist for the audit. If the operator is not finding any of these modes during the audit, either the framework is in unusually good shape, or the audit is not looking honestly enough.
